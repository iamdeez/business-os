# Spec: B2B Agency MVP

> Branch: `docs/SPEC-001-b2b-agency-mvp` | Date: 2026-06-28 | Version: v1.0.0

## 배경 및 목적

소규모 B2B 에이전시는 고객 문의, 담당자 정보, 작업 파일과 전달 이력을 이메일·메신저·스프레드시트에 나누어 관리한다. 이로 인해 문의 누락, 고객 정보 중복, 파일 전달 지연과 현재 상태 파악의 어려움이 발생한다.

첫 포트폴리오 업종을 B2B 에이전시로 확정하고, 문의 접수부터 고객 관리와 파일 전달까지 하나의 Business OS에서 처리하는 판매 가능 MVP를 만든다. 이 spec의 목적은 2주 안에 시연 가능한 핵심 흐름을 완성하면서 향후 견적·계약·프로젝트 관리 모듈을 추가할 수 있는 tenant 기반을 마련하는 것이다.

## 사용자 스토리 (User Stories)

- **US-001**: 잠재 고객으로서, 상담을 시작하기 위해 에이전시에 프로젝트 문의를 제출하고 싶다.
- **US-002**: 에이전시 운영자로서, 허가된 사용자만 운영 화면에 접근하게 하고 싶다.
- **US-003**: 에이전시 운영자로서, 고객사와 담당자 정보를 한 곳에서 조회·등록·수정하고 싶다.
- **US-004**: 에이전시 운영자로서, 새 문의를 놓치지 않고 상태별로 처리하고 싶다.
- **US-005**: 에이전시 운영자로서, 고객에게 전달할 파일을 고객사별로 관리하고 싶다.
- **US-006**: 고객사 담당자로서, 계정 생성 없이 안전한 링크로 전달 파일을 확인하고 싶다.
- **US-007**: 에이전시 운영자로서, 문의 접수와 파일 공유 시 필요한 이메일이 발송되고 결과가 기록되길 원한다.
- **US-008**: 에이전시 운영자로서, 대시보드에서 처리해야 할 문의와 고객·파일 현황을 빠르게 파악하고 싶다.

## 기능 요구사항 (Functional Requirements)

- **FR-001**: 공개 문의 화면은 에이전시를 식별하고 회사명, 담당자명, 이메일, 연락처(선택), 프로젝트 유형, 예산 범위(선택), 희망 일정(선택), 문의 내용을 입력받아 접수해야 한다.
- **FR-002**: 문의 제출자는 접수 성공 여부를 확인할 수 있어야 하며, 동일 요청의 반복 전송으로 문의가 중복 생성되지 않아야 한다.
- **FR-003**: 운영자는 이메일과 비밀번호로 로그인·로그아웃할 수 있어야 하며, 인증되지 않은 사용자는 운영 화면에 접근할 수 없어야 한다.
- **FR-004**: 모든 운영 데이터 조회·생성·수정은 현재 사용자가 소속된 tenant 범위 안에서만 수행되어야 한다.
- **FR-005**: 운영자는 고객사의 회사명, 담당자명, 이메일, 연락처, 메모, 상태를 등록·조회·수정할 수 있어야 한다.
- **FR-006**: 운영자는 고객 목록을 회사명 또는 담당자명으로 검색하고 상태로 필터링할 수 있어야 한다.
- **FR-007**: 운영자는 문의 목록과 상세 내용을 조회하고 문의 상태를 `NEW`, `IN_PROGRESS`, `RESOLVED`로 변경할 수 있어야 한다.
- **FR-008**: 운영자는 문의를 기존 고객사에 연결하거나 새 고객사로 등록할 수 있어야 한다.
- **FR-009**: 운영자는 허용된 파일을 고객사에 연결하여 업로드하고 파일명, 크기, 유형, 등록 시각을 확인할 수 있어야 한다.
- **FR-010**: 운영자는 고객사에 전달할 파일을 선택해 만료·폐기 가능한 공유 링크를 생성할 수 있어야 한다.
- **FR-011**: 유효한 공유 링크를 가진 고객사 담당자는 선택된 파일 목록을 확인하고 다운로드할 수 있어야 한다.
- **FR-012**: 새 문의가 접수되면 운영자에게, 공유 링크가 생성되면 고객사 담당자에게 이메일을 발송하고 발송 결과를 기록해야 한다.
- **FR-013**: 대시보드는 전체 고객 수, `NEW` 문의 수, 전체 파일 수와 최근 문의를 표시해야 한다.
- **FR-014**: B2B 에이전시 데모를 위한 tenant, 운영자, 고객사, 문의와 파일 메타데이터 예시를 재현 가능하게 준비해야 한다.

## 비기능 요구사항 (Non-Functional Requirements)

- **NFR-001**: tenant 간 고객, 문의, 파일, 공유 링크와 알림 기록이 노출되지 않아야 한다.
- **NFR-002**: 비밀번호, 인증 토큰, 공유 링크 원문과 외부 서비스 키를 로그·문서·클라이언트 코드에 노출하지 않아야 한다.
- **NFR-003**: 파일 업로드와 다운로드는 애플리케이션 서버가 파일 본문을 중계하지 않아야 한다.
- **NFR-004**: 이메일과 파일 외부 연동 실패가 고객·문의 데이터의 저장 결과를 되돌리거나 손상시키지 않아야 한다.
- **NFR-005**: 운영 화면과 공개 화면은 모바일과 데스크톱에서 핵심 흐름을 수행할 수 있어야 한다.
- **NFR-006**: 폼 입력 요소는 레이블, 키보드 탐색, 오류 설명을 제공해야 한다.
- **NFR-007**: 고객·문의 목록은 페이지 단위로 조회하여 데이터 증가가 한 번의 무제한 조회로 이어지지 않아야 한다.
- **NFR-008**: 모든 기능 요구사항은 자동화된 단위·통합 또는 E2E 검증 시나리오와 연결되어야 한다.

## 수용 기준 (Acceptance Criteria)

- **SC-001** (`FR-001`, `FR-002` 관련): 유효한 공개 문의를 제출하면 성공 화면이 표시되고 운영자 문의함에 정확히 한 건이 `NEW`로 생성된다.
- **SC-002** (`FR-001`, `FR-002` 관련): 필수값 누락, 잘못된 이메일 또는 동일 요청 재전송은 새 문의를 만들지 않고 이해 가능한 오류를 반환한다.
- **SC-003** (`FR-003` 관련): 유효한 운영자는 로그인 후 대시보드에 접근하고 로그아웃할 수 있으며, 비인증 접근은 로그인 화면으로 이동한다.
- **SC-004** (`FR-004` 관련): 서로 다른 두 tenant의 사용자는 동일 식별자를 추측해도 상대 tenant의 고객·문의·파일·공유 링크를 읽거나 변경할 수 없다.
- **SC-005** (`FR-005`, `FR-006` 관련): 운영자는 고객사를 생성·수정하고 회사명·담당자명 검색 및 상태 필터 결과를 확인할 수 있다.
- **SC-006** (`FR-007`, `FR-008` 관련): 운영자는 문의 상태를 허용된 값으로 변경하고 문의를 기존 또는 신규 고객사와 연결할 수 있다.
- **SC-007** (`FR-009` 관련): 허용된 파일은 고객사 경로에 업로드되고 메타데이터가 저장되며, 금지된 유형·초과 크기·tenant 불일치는 거부된다.
- **SC-008** (`FR-010`, `FR-011` 관련): 공유 링크는 선택 파일만 보여주며 만료 또는 폐기 후에는 파일 목록과 다운로드 URL을 제공하지 않는다.
- **SC-009** (`FR-012` 관련): 문의 접수와 파일 공유 이메일의 성공·실패 결과가 각각 추적 가능하게 기록되고 중복 요청은 중복 발송되지 않는다.
- **SC-010** (`FR-013` 관련): 대시보드 집계가 저장된 고객·문의·파일 데이터와 일치하고 최근 문의가 최신순으로 표시된다.
- **SC-011** (`FR-014` 관련): 한 번의 명령으로 동일한 데모 기준 데이터를 생성할 수 있고, 반복 실행해도 중복 데이터가 누적되지 않는다.
- **SC-012** (`NFR-005`, `NFR-006` 관련): 390px 모바일 화면과 데스크톱에서 로그인, 문의 제출, 고객 조회, 파일 공유 흐름이 가로 스크롤 없이 완료되고 핵심 폼을 키보드로 조작할 수 있다.
- **SC-013** (`NFR-008` 관련): lint, typecheck, 단위·통합 테스트와 핵심 E2E 시나리오가 CI에서 통과한다.

## 범위 외 (Out of Scope)

- 견적서 생성·승인, 계약, 전자서명과 결제·청구 기능
- 프로젝트 칸반, 단계별 진행 상태와 고객 포털
- 예약·일정, 출결과 카카오 알림톡
- AI 문의 분류, 답변 초안과 챗봇
- 운영자 초대, 조직 관리 UI와 세분화된 권한 편집
- 고객의 상시 계정과 비밀번호 기반 고객 포털
- 파일 버전 관리, 온라인 미리보기와 대용량 미디어 스트리밍
- Slack·Notion에 고객 업무 이벤트를 자동 전송하는 제품 기능
