## 한 줄 소개

**B2B 에이전시를 위한 멀티테넌트 SaaS를 기획·설계·구현하고 실제 프로덕션(Fly.io)까지 배포한 풀사이클 프로젝트.** 가입하면 팀 전용 워크스페이스가 자동 생성되고, 고객 문의 접수 → CRM → 파일 공유 → 이메일 알림 → 현황 대시보드가 하나의 흐름으로 연결됩니다.

## 프로젝트 개요

|---|---|

| 코드 | https://github.com/iamdeez/business-os |

## 해결한 문제

소규모 B2B 에이전시는 **문의 응대 → 고객 등록 → 파일 전달 → 진행 상황 확인**을 이메일·엑셀·메신저에 흩어서 반복합니다. 도구가 분산돼 있어 누락·중복·컨텍스트 스위칭 비용이 큽니다.

Business OS는 이 반복 흐름을 **한 워크스페이스**로 통합해, 각 에이전시가 가입 즉시 자기 팀 전용 공간에서 고객 운영 업무를 처리하도록 합니다.

## 주요 기능

- **자가 가입 + 워크스페이스 자동 프로비저닝** — 가입 시 Tenant + OWNER 권한을 트랜잭션으로 생성
- **공개 문의 접수** — 워크스페이스별 공개 폼(`/inquiry/{slug}`) → 관리자 인박스 (honeypot·멱등)
- **고객 CRM** — tenant 범위 CRUD·검색·상태 필터·페이지네이션
- **파일 업로드·공유** — 브라우저가 스토리지에 **직접(presigned) 업로드**, 만료·폐기 가능한 공유 링크
- **이메일 알림** — 문의 접수·파일 공유 시 발송, 발송 로그 + 이중 멱등
- **현황 대시보드** — 고객·문의·파일 집계 + 최근 활동 + 공개 문의 링크

## 기술 스택 & 아키텍처

**스택**: Next.js 16(App Router, Server Actions) · TypeScript(strict) · Tailwind 4 · Prisma 7 + PostgreSQL(Supabase) · Better Auth · S3 호환 스토리지(Cloudflare R2) · Resend · Vitest + Playwright · GitHub Actions · Docker + Fly.io

**레이어드 아키텍처** — UI → 도메인 모듈(schema·repository·service·actions) → lib(외부 경계). UI는 외부 SDK를 직접 호출하지 않고, 순수 정책 모듈은 `server-only` 의존 없이 단위 테스트합니다.

## 기술적 도전과 해결

### 1. 멀티테넌트 데이터 격리 + 자동 프로비저닝
모든 데이터 접근을 `tenantId` 범위로 강제해 교차 테넌트 접근을 차단했습니다. 자가 가입 SaaS로 전환하며 "가입은 됐는데 워크스페이스가 없어 진입 불가"한 공백을 발견 → Better Auth의 `databaseHooks`에서 Tenant + OWNER 멤버십을 **트랜잭션으로 생성(멱등·slug 충돌 재시도)**하도록 구현하고, 라이브에서 가입→격리된 워크스페이스 생성을 검증했습니다.

### 2. 서버를 거치지 않는 파일 전송
파일 본문이 앱 서버를 통과하면 메모리·대역폭 비용이 큽니다. presigned URL을 발급해 **브라우저 ↔ 스토리지 직접 전송**하고, 서버는 정책 검증과 업로드 확정(HeadObject)만 담당하도록 설계했습니다. 비용 절감을 위해 AWS S3 → Cloudflare R2로 전환하면서 **최신 aws-sdk의 checksum 헤더가 R2 presigned 서명을 깨뜨리는** 이슈를 옵션 조정으로 해결했습니다.

### 3. "로컬은 되는데 배포만 실패"
배포 후 `P1001 DatabaseNotReachable`. Prisma CLI는 SSL을 자동 적용해 로컬 마이그레이션은 통과했지만, 앱 런타임의 raw `pg.Pool`은 관리형 DB에 필요한 **SSL을 켜지 않아** 실패했습니다. 비-로컬 호스트에 SSL을 활성화해 해결. "환경별 차이로만 재현되는 버그"를 계층별로 좁혀간 사례입니다.

### 4. 테스트가 잡은 회귀
대시보드에 UI를 추가하자 모바일(390px)에서 가로 스크롤이 생겼고, **Playwright 반응형 E2E가 CI에서 이를 사전 차단**했습니다. flexbox의 `min-w-0`로 해결. 자동화 테스트가 실제 회귀를 막은 경험으로, 테스트 투자 대비 효용을 체감했습니다.

## 성과

- 기획부터 **프로덕션 배포까지 1인 풀사이클** 완주 (라이브 SaaS 운영 중)
- **87개 자동화 테스트**(단위·통합·E2E) + GitHub Actions CI(PostgreSQL service + Chromium E2E)
- **presigned 직접 업로드·멱등성 3종·안전한 공유 링크** 등 실무 패턴 적용
- Docker 멀티스테이지(~80MB) + Prisma 마이그레이션 파이프라인으로 재현 가능한 배포

## 배운 점

- 멀티테넌시는 "쿼리 격리"만이 아니라 **온보딩(프로비저닝)까지 설계**해야 완성된다.
- 환경 의존 버그(SSL·CORS·빌드 인라인 env)는 **계층을 분리해 재현 조건을 좁히는 것**이 핵심.
- E2E·CI는 비용이 아니라, 회귀를 사람보다 먼저 잡는 **레버리지**다.
