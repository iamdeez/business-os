# Project Context

> 현재 코드베이스의 사실을 기록한다. 미래 계획은 `.claude/idea.md` 또는 spec에 둔다.

## 1. 프로젝트 개요

- **프로젝트명**: Business OS (`kmong` 작업 폴더)
- **목적**: 반복적인 고객 관리·문의·파일·예약 업무를 통합하는 크몽 판매용 비즈니스 운영 자동화 솔루션 기획
- **현재 버전**: `[미파악 - 구현 전]`
- **현재 기술 스택**: `[미파악 - 소스 코드와 패키지 설정 없음]`
- **기획된 기술 스택**: Next.js 15, TypeScript, PostgreSQL, Prisma 등이며 확정·구현 전 상태

## 2. 프로젝트 구조

```text
kmong/
├── AGENTS.md
├── CONTRIBUTING.md          # Git·GitHub 협업 규칙
├── README.md                # 프로젝트 진입점
├── .github/                 # PR 템플릿, CODEOWNERS, 정책 검사
├── .claude/
│   └── idea.md              # 제품·아키텍처 기획 원문
└── .Codex/
    ├── constitution.md      # 불변 원칙
    ├── context.md           # 현재 코드 상태
    ├── infra.md             # 현재 운영 상태
    ├── integrations.md      # GitHub·Slack·Codex 연결 정보
    ├── project.md           # 프로젝트 관리 허브
    └── templates/           # 반복 업무 템플릿
```

- 애플리케이션 소스와 패키지 설정은 아직 존재하지 않는다.
- 레이어 구조와 핵심 모듈은 구현 후 기록한다.

## 3. 이벤트 및 데이터 흐름

- `[미파악 - 구현 전]`
- `.claude/idea.md`에 문의, 고객, 파일, 알림 등 계획 흐름이 있으나 현재 구현된 흐름은 없다.

## 4. 도메인 모델

- 구현된 엔티티는 없다.
- `Tenant`, `User`, `Customer`, `Inquiry`, `Reservation`, `Project`, `FileItem`, `NotificationLog`는 기획 초안이며 현재 코드 상태로 간주하지 않는다.

## 5. 도메인 용어 사전

| 용어 | 정의 | 사용 금지 동의어 |
|---|---|---|
| Business OS | 고객사의 반복 운영 업무를 통합·자동화하는 시스템 | 단순 홈페이지, 단순 챗봇 |
| tenant | 시스템을 독립적으로 사용하는 고객사 단위 | 고객 개인 |
| Core | 고객·업종에 공통으로 재사용하는 기반 기능 묶음 | 고객별 복제본 |
| 모듈 | CRM, 문의, 예약 등 선택적으로 조합하는 기능 단위 | 페이지 |
